## 프로젝트 개요

팀 단위로 예약을 관리하고, 날짜별 예약 현황을 확인할 수 있는 웹 서비스입니다.  
React Calendar를 커스텀하여 날짜별 예약 팀을 조회할 수 있도록 구현했으며,  
프론트엔드 개발 전반과 기본적인 디자인 구성까지 담당했습니다.  
React와 Tailwind CSS를 처음 사용해보며 실제 프로젝트 구조를 경험한 첫 번째 실전형 프로젝트였습니다.

---

## 개발 기간

- 전체 개발: 2024.11.21 - 2024.12.15

## 프로젝트 스토리

React와 JavaScript를 중심으로 예약 신청 폼과 달력 UI를 직접 구성했습니다.  
폼 작성 과정에서는 `useReducer`를 사용하여 입력 상태를 관리했고,  
비즈니스 로직은 **커스텀 훅(Custom Hook)** 형태로 분리하여 재사용성을 높이고자 했습니다.

Tailwind CSS를 처음 도입하면서 클래스 네이밍보다 빠른 프로토타이핑에 집중할 수 있었고,  
짧은 기간 내에 작동 가능한 예약 관리 기능을 완성했습니다.

---

## 기술적 포인트

### 1. React Calendar 커스텀

- 날짜별 예약 현황을 시각적으로 표시
- 클릭 시 해당 날짜의 예약 팀 목록 표시

### 2. 예약 신청 로직

- `useReducer`를 이용해 폼 상태 관리
- Axios를 사용한 API 통신 및 예약 요청 처리
- 주요 로직을 커스텀 훅으로 분리하여 재사용 가능하도록 설계

### 3. 스타일링

- Tailwind CSS로 빠른 UI 제작 및 반응형 적용
- 간단한 컬러 팔레트와 레이아웃 중심의 디자인

## 결과 및 개선

### 당시 구현 방식의 한계

- `axios.then().catch().finally()` 체인으로 인한 **복잡한 오류 처리 구조**
- **로딩 상태 미고려**, 전반적으로 일관되지 않은 상태 관리
- 토큰 삽입 등 인증 관련 로직이 분리되지 않아 유지보수가 어려움
- 커스텀 훅 분리 시 **함수 네이밍 및 역할 명확성 부족**

### 개선 아이디어

- Axios Wrapper 및 에러 핸들러 통합
- `react-query` 등 상태 기반의 데이터 캐싱 도입
- 명확한 훅 네이밍 및 역할 분리로 가독성 향상
